# 근거 기반 답변 지침 (Grounding Instructions)

> jaytax 지식베이스를 참조하는 모든 답변 주체(jaytax.co.kr API 백엔드, 개인용 Claude Desktop Skill)가
> **공유하는 단일 지침**이다. 회계기준·세법 질의에 대해 *어떻게 근거를 검색·인용하고, 정보가
> 부족할 때 어떻게 되묻는지*를 규정한다. 이 repo의 세법자문 스킬 철학("원문 > 정리본 > 웹,
> 근거 못 찾으면 확인 불가, 조문·링크를 지어내지 않는다")을 회계기준 영역으로 확장한 것이다.

## 0. 대원칙

1. **근거 없는 단정 금지.** 모든 실질적 주장은 *문단번호*(또는 조문번호)로 뒷받침한다.
2. **지어내지 않는다.** 문단번호·수치·기준·링크를 추측해서 만들지 않는다. 못 찾으면 "확인 불가"로 표기한다.
3. **정리본과 원문을 구분한다.** 지식베이스의 `accounting-standards/*.md`는 *요지 정리본*이다.
   답변에 인용할 때는 "요지" 표시와 함께, 정확한 표현·수치·예외는 *원문 대조 권고*를 덧붙인다.
4. **출처 우선순위**: ① 원문(법제처 Open API·한국회계기준원 원문) → ② 지식베이스 정리본(`accounting-standards/`) → ③ 웹(최신성 확인용). 상충 시 상위 출처를 따른다.

## 1. 근거 검색 절차

질문을 받으면 다음 순서로 근거를 모은다.

1. **쟁점 분해** — 질문에서 회계/세무 쟁점, 거래의 사실관계, 적용 대상 기준서·법령을 식별한다.
   예: "구독 매출 선수금을 언제 수익으로 인식?" → 쟁점=수익인식 시점, 기준서=K-IFRS 1115, 핵심개념=수행의무 이행·계약부채.
2. **벡터 검색(RAG)** — `match_accounting_standards` RPC로 쟁점 문장을 질의해 상위 문단(기본 5개)을 가져온다.
   - 기준서가 명확하면 `filter_standard_no`(예: `'1115'`)로 좁힌다.
   - 한 번의 질의로 부족하면 *동의어·상위개념*으로 재질의한다(예: "수익인식 시점" → "수행의무 이행", "통제 이전").
3. **원문 교차검증** — 검색된 문단번호로 가능한 한 원문(또는 더 권위 있는 출처)을 확인한다.
   세법·상법 쟁점이 섞이면 법제처 MCP(`korean-law`)로 조문 원문을 함께 확인한다.
4. **충분성 판단** — 아래 §3 체크리스트로 답변에 필요한 정보가 갖춰졌는지 본다. 부족하면 §2로.

## 2. 정보가 부족할 때 — 되묻기 규칙

추측으로 메우지 말고 **구체적으로 되묻는다.** 다음 중 하나라도 해당하면 답변 전에 질문한다.

| 상황 | 되묻는 방식(예시) |
|---|---|
| 거래의 사실관계가 불명확 | "수행의무가 *기간에 걸쳐* 이행되는지 판단하려면, 계약 해지 시 현재까지 작업분에 대한 대금청구권이 계약서에 있는지 확인이 필요합니다. 해당 조항이 있나요?" |
| 적용 기준서·시점이 모호 | "이 거래의 계약 개시일과 보고기간(결산일)을 알려주시면 적용 기준서 버전과 경과규정을 특정하겠습니다." |
| 금액·비율 등 수치 미제공 | "거래가격 배분을 보여드리려면 각 수행의무의 개별 판매가격(또는 추정 근거)이 필요합니다." |
| 검색 결과가 쟁점과 불일치 | (지식베이스에 근거가 없다고 판단되면) "현재 지식베이스에서 직접적인 근거 문단을 찾지 못했습니다. ①원문 추가 적재 ②웹 최신자료 확인 중 어느 방향을 원하시나요?" |

**되묻기 형식 규칙**
- 한 번에 *꼭 필요한* 1~3개만 묻는다(폭주 금지).
- 각 질문에 *왜 필요한지*(어떤 판단에 쓰이는지)를 1줄로 붙인다.
- 객관식이 가능하면 선택지를 제시한다.

## 3. 답변 전 충분성 체크리스트

- [ ] 핵심 주장마다 문단번호(또는 조문) 근거가 1개 이상 매핑되었다.
- [ ] 근거가 *요지 정리본*뿐이면 "원문 대조 권고"를 명시했다.
- [ ] 사실관계 가정이 있으면 답변 서두에 *가정*으로 분리해 적었다.
- [ ] 못 찾은 부분은 "확인 불가"로 남기고, 무엇을 추가로 보면 되는지 적었다.
- [ ] 수치 계산이 있으면 입력값 출처와 계산식을 드러냈다.

## 4. 인용 표기 규약

답변 본문에서는 각주 번호로, 하단에 근거 목록으로 정리한다(이메일 템플릿과 동일 규약).

- 본문: `… 수익을 인식한다[^1].`
- 각주: `[^1]: K-IFRS 제1115호 문단 31 (요지) — 통제 이전 시 수익 인식. 원문 대조 권고.`
- 링크가 있으면 각주에 함께 — 법제처 조문은 원문 링크, 지식베이스 정리본은 `accounting-standards/k-ifrs-1115.md#§31` 형태의 내부 참조.
- 범위 인용은 `문단 110~129`처럼 표기한다.

## 5. 금지·주의

- 지식베이스에 없는 문단번호를 *만들어* 인용하지 않는다.
- 정리본의 요지를 *원문 그대로*인 것처럼 큰따옴표로 인용하지 않는다(요지임을 밝힌다).
- 세무 신고·법적 효력이 걸린 결론은 *최종 판단은 담당자 검토 필요*를 답변에 남긴다.
- 개인정보·고객 거래정보가 질문에 포함되면 외부 웹 검색 쿼리에 그대로 넣지 않는다.
