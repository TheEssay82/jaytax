# 개발노트 / 변경이력 (CHANGELOG)

인덕회계법인 **세무조정수수료 관리시스템** — 구 단일 HTML 앱(`ver.4.6`)을 React + TypeScript + Supabase 웹서비스로 전환한 기록.

## 버전 관리 방식
- **앱 버전**: `package.json`의 `version` → 빌드 시 화면 헤더에 `v1.0.0` 형태로 표시 (좌상단 제목 옆).
- **수정 이력**: 모든 변경은 git 커밋으로 관리. `git log` 가 1차 기록이며, 이 파일이 사람이 읽기 쉬운 요약본.
- **설정(수수료) 버전**: 위 앱 버전과 별개. 설정 탭에서 누진율·가산율 묶음을 버전으로 저장/적용 (예: `기본_260626`).
- **버전 번호 규칙(권장, SemVer)**: `MAJOR.MINOR.PATCH`
  - PATCH(0.0.x): 버그 수정·문구 변경
  - MINOR(0.x.0): 기능 추가(하위호환)
  - MAJOR(x.0.0): 큰 구조 변경
  - 버전 올릴 때: `package.json` version 수정 → 이 파일에 항목 추가 → 커밋.

## [1.0.0] — 2026-06-27 · 웹 전환 완료 (구 HTML ver 4.6 후속)

원본 HTML 앱의 7개 탭 전체를 React + Supabase 로 포팅하고 실데이터를 이관한 첫 정식 버전.

### 기반
- Vite + React 19 + TypeScript, 원본 디자인 CSS 유지(`legacy.css`)
- Supabase 인증(직원 로그인) + Postgres + RLS(익명 차단), git 버전관리 도입
- 수수료 계산 로직(`calc.ts`) 포팅

### 탭 기능 (7/7)
- **청구서 작성**: 6단계 위저드(거래처 선택 → 기본정보 → 업무량 → 수수료 확인 → 금액 조정 → 청구서). 거래처 프리필, 납부기한 자동, 도움말 툴팁, 전년/과거 비교, 저장(거래처 매출·담당자 자동갱신), 인쇄/PDF.
- **거래처 관리**: CRUD, 검색·필터·정렬, 연도별 매출/성실신고, 매출액 일괄입력·Excel 붙여넣기, 엑셀 양식 다운로드/업로드(사업자번호 upsert).
- **청구대상**: 연도별 법인/개인 확정·일괄확정.
- **청구기록**: 목록·필터·정렬·합계, 상세 펼침, 비교·수정·삭제.
- **통계**: 담당자별 재무·업무량 집계.
- **업데이트요청**: 게시판 + 상태 + 댓글 (원본 localStorage 저장 버그 → DB 영구저장으로 해결).
- **설정**: 수수료 설정 다중 버전 + 전역 활성버전 선택, DEFAULT 고정 선택지, 업무항목 설명 편집.

### 데이터 이관
- 구 HTML 앱 JSON 백업 → Supabase 마이그레이션: 거래처 79 · 청구기록 134(2024:73, 2025:61) · 청구대상. 거래처 ID(타임스탬프) → UUID 재매핑.

### 주요 수정·개선
- 청구현황(isBilled)·신규 판정을 **ID + 회사명 매칭**으로 보강 (거래처 ID 변경에도 안 깨짐).
- 거래처 선택 화면: 청구대상을 배타적 필터 → **상단 우선 정렬**(전체 거래처 표시).
- 연도별 활성상태 구분: **거래전(신규 이전)** / **상실(거래종료)** 배지 + 미청구 필터 제외.
- 청구서 구조 개선: 가산세를 ⑦에서 분리해 **⑨ 가산금액**(C 이후 가산)으로, 부가세 ⑩으로 재번호. ⑧ 할인·⑨ 가산 비고에 사유 연동.
- 도움말 툴팁 표시·설정 반영, 인쇄 시 불필요 요소 숨김 등.

> 상세 커밋 단위 이력은 `git log` 참조 (2026-06-26 ~ 2026-06-27, 24개 커밋).
